# PRD — Ya Korea Hostel 실시간 채팅 시스템

## Why (목적)
호스텔 직원이 반복 질문에 응대하는 시간을 줄이고, 손님이 24시간 즉시 답변을 받을 수 있게 한다.

## Who (타겟 사용자)
- **손님(Guest)**: 호스텔 키오스크 또는 웹사이트를 이용하는 외국인/내국인 투숙객
- **매니저(Manager)**: 관리자 페이지에서 채팅을 모니터링하고 응대하는 호스텔 직원

## 핵심 유저 플로우

### 손님 플로우
1. 키오스크/웹페이지 접속 → 우측 하단 채팅 위젯 아이콘 클릭
2. 채팅창 열림 → 질문 입력
3. **자동응답 엔진이 FAQ 매칭 시도**
   - 유사도 ≥ 0.8: 자동응답 즉시 전송
   - 유사도 0.5~0.8: 후보 답변 제시 ("이런 내용을 찾으시나요?")
   - 유사도 < 0.5: "매니저 연결" 버튼 표시
4. 매니저 연결 선택 시 → 실시간 채팅 전환

### 매니저 플로우
1. 관리자 페이지 로그인
2. 대화방 목록에서 연결 요청 확인
3. 방 입장(join_room) → 손님과 실시간 채팅
4. 대화 종료 처리

## 핵심 기능 (In Scope)
- [ ] FAQ 자동응답 (Jaccard 유사도)
- [ ] Socket.IO 실시간 채팅 (손님 ↔ 매니저)
- [ ] 채팅 위젯 (Shadow DOM, 프레임워크 없음)
- [ ] 관리자 페이지 (Next.js)
- [ ] 채팅 히스토리 저장 (Supabase)
- [ ] Railway 서버 + Vercel 관리자 배포

## Out of Scope (이번 버전에 개발하지 않음)
- 다국어 번역 기능
- 파일/이미지 전송
- 손님 회원가입/로그인
- 챗봇 AI 연동 (GPT 등)
- 푸시 알림 (모바일)
- 매니저 간 내부 채팅
- 통계/대시보드 페이지

## 성공 기준
- 자동응답 정확도 80% 이상 (테스트 질문 세트 기준)
- 메시지 전달 지연 500ms 이하
- 위젯 → 관리자 E2E 테스트 통과
