# Decisions — 기술 의사결정 기록 (ADR)

AI가 컴팩션(기억 상실) 후에도 과거 결정을 반복하지 않도록 기록.

---

## ADR-001: 위젯 — 순수 JS + Shadow DOM (프레임워크 없음)
**결정**: React/Vue 없이 순수 JS로 위젯 구현
**이유**: 호스텔 키오스크/웹 어디서든 script 태그 하나로 삽입 가능해야 함. 프레임워크 번들이 호스트 페이지와 충돌 위험 있음. Shadow DOM으로 CSS 격리.
**거부된 대안**: React 위젯 — 호스트 페이지 React 버전 충돌 가능성

---

## ADR-002: 서버 주소 감지 — script src URL 자동 파싱
**결정**: 위젯 JS 파일의 script src 속성에서 서버 URL 자동 추출
**이유**: 위젯을 키오스크에 설치 시 서버 URL을 하드코딩하면 배포 환경 변경 시 위젯 재배포 필요. src URL 기반 자동 감지로 유연성 확보.
**구현 패턴**:
```js
const scriptSrc = document.currentScript.src;
const serverUrl = new URL(scriptSrc).origin;
```

## ADR-003: 매니저 room 조인 — 명시적 이벤트 방식
**결정**: 매니저가 채팅방 진입 시 `manager:join_room` 이벤트를 반드시 전송
**이유**: 이전 구현에서 매니저가 Socket.IO room에 조인하지 않아 메시지가 전달되지 않는 버그 발생. 명시적 조인 이벤트로 room 멤버십 보장.
**절대 금지**: 매니저 소켓 연결만으로 모든 방의 메시지를 수신하게 하는 broadcast 방식

## ADR-004: FAQ 매칭 — Jaccard 유사도
**결정**: Jaccard 유사도로 FAQ 매칭
**이유**: 외부 AI API 의존 없이 서버 내에서 빠르게 처리 가능. 호스텔 FAQ는 패턴이 한정적이라 단순 유사도로 충분.
**임계값**: ≥0.8 자동응답, 0.5~0.8 후보 제시, <0.5 매니저 연결
**거부된 대안**: GPT API — 비용 발생, 레이턴시 증가, 오버엔지니어링

## ADR-005: DB — Supabase (스키마 변경 금지)
**결정**: 기존 Supabase 스키마 그대로 사용
**이유**: 스키마가 이미 확정되어 있으며, 변경 시 기존 데이터 마이그레이션 필요.
**주의**: AI가 임의로 컬럼 추가/변경하는 행위 금지. 스키마 변경이 필요하면 반드시 사용자에게 먼저 질문.

## ADR-006: 재연결 처리 — guest:join 자동 재전송
**결정**: Socket.IO 재연결 이벤트 시 로컬 스토리지의 roomId로 guest:join 자동 재전송
**이유**: 네트워크 끊김 후 재연결 시 room 멤버십이 초기화되어 메시지 수신 불가 버그 발생 이력 있음.
**구현 위치**: 위젯 클라이언트 socket reconnect 이벤트 핸들러
